# 里程碑 6 设计说明：NTFS 快速层（MFT 直读 + USN Journal）

目标一句话：在 NTFS 卷上，首次建文件名索引从"分钟级爬目录"变成"秒级读账本"，
错过的变更从"全盘 mtime 对账"变成"读日志流水补账"。Everything 的两板斧,
接进我们已有的三层索引架构。

## 一、与现有架构的关系

里程碑 3 已经留好了插座：事件源只需产出 `WatchEvent`（Upsert / Remove /
RemoveDir / Rename）四种归一事件，防抖、合并、水位、增量更新全部复用。
本里程碑做三件事：

1. **MFT 快速枚举**——初次扫描不走 walkdir 爬目录，直接读 NTFS 的主文件表；
2. **USN 事件源**——运行期监听从 notify 换成读 USN Journal（notify 保留为降级路径）；
3. **USN 游标补账**——启动对账优先按游标回放日志，mtime 全扫降为兜底。

```
                     ┌── 有管理员权限 + NTFS ──▶ MFT 枚举 + USN 事件源 + 游标补账
  每个监听根按卷判定──┤
                     └── 否则 ──────────────────▶ walkdir + notify + mtime 对账（现状）
```

降级判定按卷、在运行时做，两条路径产出的事件和文档完全一致——上层感知不到差别。
诚实降级：落到哪条路径、为什么，写日志（材质降级那次的教训）。

## 二、MFT 快速枚举

- 用卷句柄发 USN 枚举控制码（FSCTL_ENUM_USN_DATA）遍历全卷文件记录——这是
  Everything 同款技术：不打开任何文件、不走目录树，顺序读出每条记录的
  文件参照号（FRN）、父 FRN、文件名。
- 内存里按 FRN → (父 FRN, 名字) 建表，从叶子回溯拼出完整路径；只保留落在
  已注册监听根内的条目（全盘模式本里程碑不做）。
- 拼出来的清单喂给现有索引管线。MFT 记录不含可靠的 mtime/size，
  走"文件名先入索引 + 内容抽取后台补"的既有节奏——这正是三层索引
  "逐层变重"设计预留的形态。
- 需要管理员权限拿卷句柄。没有就整条路径不可用，静默走 walkdir，日志记一行。

## 三、USN 事件源

- 按卷起一个读取线程，FSCTL_READ_USN_JOURNAL 阻塞式拉取记录，翻译成
  WatchEvent 喂进现有防抖队列：
  - 创建/写入/属性变更 → Upsert
  - 删除 → Remove（目录记录带目录标志 → RemoveDir）
  - 重命名是**两条记录**（旧名 + 新名，同一 FRN）：配对成 Rename{from,to}；
    配不上对的单边按 in_scope 规则自然退化（防抖队列已处理）
- **已知的坑，设计期就写进测试清单**：同一 FRN 上"重命名后紧跟删除"这类
  快速连续操作，记录配对状态机容易吞事件——单测要覆盖乱序/交错序列。
- USN 记录给的是 FRN + 文件名，不是全路径：维护 FRN → 路径 的缓存表
  （MFT 枚举时顺手建好，收到目录事件时增量维护）。缓存 miss 时用
  FRN 打开句柄反查路径兜底。

## 四、游标补账

- meta.json 每卷存两个数：Journal ID + 下一个 USN 位置，每批 commit 成功后更新。
- 启动时：Journal ID 对得上且游标仍在日志保留范围内 → 从游标回放到当前,
  秒级追平;Journal 被重建/回绕（ID 变了或游标过期）→ 退回 mtime 全扫对账。
- 游标持久化与批量 commit 的原子性：先 commit 索引、后写游标——宁可重放一批
  （Upsert 幂等），不可漏一批。

## 五、性能预算

| 指标 | 预算 |
|------|------|
| MFT 枚举 100 万条记录 | < 5s |
| 文件名层可搜（10 万文件的根） | < 10s（内容后台继续补） |
| USN 游标回放 10 万条变更 | < 10s |
| USN 读取线程常驻内存（含 FRN 路径表） | < 80MB / 百万条目 |

## 六、验收清单

1. 管理员运行，对一个 10 万文件的 NTFS 根建索引：文件名层 10 秒内可搜。
2. 非管理员运行同样操作：自动走 walkdir 路径，功能等价，日志明确说明降级原因。
3. 停掉 dowse，改/删/增/重命名各若干文件，重启：USN 回放追平，
   不触发全盘 mtime 扫描（日志验证）。
4. 快速连续"改名→删除"同一文件：索引终态正确（旧名新名都搜不到）。
5. FAT32/exFAT 卷（U 盘）上的监听根：自动落 notify 路径，不报错。
6. 游标过期场景（人为改 meta.json 的 USN 为过期值）：正确退回 mtime 对账。

## 七、明确不做（本里程碑）

- 全盘文件名模式（不注册根、直接搜整个 C 盘）——留给以后的"Everything 模式"
  产品决策，本里程碑 MFT/USN 只服务已注册的监听根
- 非 NTFS 文件系统的任何快速路径
- ReFS 支持（它有自己的变更日志,几乎没人在桌面用,不值当）
