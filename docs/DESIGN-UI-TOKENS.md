# Raycast 设计语言抄写：token 表

这份文档只抄"设计规范"：配色层次、圆角刻度、间距节奏、字号字重、动效时长。
**不抄素材**：Raycast 的 logo、图标、品牌红、产品文案一概不进 dowse。
dowse 自己的身份保留：错位圆图标、淡水蓝 `#7FCBE8` accent、Inter + MiSans 字体栈。

截图来源：https://www.raycast.com 、 https://www.raycast.com/windows 、
https://www.raycast.com/store 、 https://www.raycast.com/ai （产品 UI 演示区），
存于本机暂存目录，未入库（截图本身是参考素材，不是交付物）。

## 一、面板底色与透明度层次

Raycast 的窗口是深色磨砂玻璃：近黑但不是纯黑，边缘有一圈极细的高光描边，
让玻璃面板在深色背景上"浮"起来，而不是一块死板的矩形。

| 层次 | 观感 | dowse 现状（改前） | dowse 改后 |
|------|------|------|------|
| 暗色玻璃底衣 | 深灰黑，透光但不发灰 | `rgba(20,20,24,0.55)` | `rgba(18,18,22,0.32)` |
| 纯色降级底 | 深灰，不分明暗 | `#1c1c20` | `#1c1c20`（不变，已经对） |
| 面板描边 | 1px，白色极低不透明度，模拟高光 | `rgba(255,255,255,0.09)` | `rgba(255,255,255,0.10)`（微调，其余不变） |
| 亮色玻璃底衣 | 白色磨砂，同样要透 | `rgba(255,255,255,0.65)` | `rgba(255,255,255,0.42)` |

**关键结论**：上一版 alpha 0.55 是方向错了，那是"磨砂玻璃上再压一层近乎不透明的漆"，
背景基本透不进来，糊成一片死灰。Raycast 的玻璃感来自高斯模糊本身做功（这里由 DWM
Acrylic 承担），前景色调只是极薄的一层中性滤镜，alpha 通常在 0.25～0.4 这个区间。

## 二、圆角刻度

Raycast 三级圆角体系，越小的元素圆角越小，避免"到处都是一样的大圆角"显得廉价：

| 元素 | 圆角 |
|------|------|
| 窗口本体 | 12～14px（截图里目测偏大，比系统默认 Win11 8px 更圆润） |
| 列表行选中态色块 | 8px |
| kbd 键帽 | 4～5px（接近直角，故意跟窗口圆角拉开差距） |
| 弹出卡片/下拉（如 AI 页 Preset 面板） | 10px |

dowse 现状：`--radius-window: 8px`（对齐 Win11 DWM 裁切，不能照抄 Raycast 的更大圆角，
这是 Win11 原生玻璃的技术约束，保留）；`--radius-row: 9px`、`--radius-chip: 5px` 已经很接近
Raycast 刻度，本次不动。

## 三、间距节奏

| 位置 | 目测值 |
|------|------|
| 面板整体内边距（列表区） | 6～8px |
| 列表行内边距 | 上下 8～9px，左右 10～12px |
| 行内图标—文字间距 | 10～12px |
| 行标题—副标题垂直间距 | 2～3px（几乎贴着，靠字号/字重区分层级而不是靠留白） |
| 区块（Favourites / Suggestions）之间 | 12～16px + 一个大写小字 label |
| 输入框内边距 | 左右 20px 左右，上下 14～16px |
| 底部操作条高度 | 40～44px |

dowse 现状基本已经落在这个区间（`.search-row padding: 14px 20px`、`.row padding: 9px 12px`、
`ShortcutBar height: 32px`）。底部操作条偏矮，本次从 32px 提到 40px，更接近 Raycast 的呼吸感。

## 四、字号层级与字重

| 场景 | 字号 | 字重 |
|------|------|------|
| 输入框 | 16～18px | Regular 400，placeholder 用低不透明度而非更浅的灰 |
| 列表行标题 | 13～14px | Medium 500（不是 Bold/600，Raycast 的标题比想象中轻） |
| 列表行副文字/路径 | 11～12px | Regular 400，纯靠颜色（不透明度）拉开层级 |
| 右侧类型提示/分类标签 | 11～12px | Regular 400，颜色更淡一档 |
| kbd 键帽文字 | 10～11px | Medium 500 |
| 区块大写 label（"Favourites"） | 11px | Medium 500，字间距加宽、颜色很淡 |

**关键结论（对应"黑黑的"反馈）**：Raycast 的正文文字不是纯白/纯黑，而是白色降到
~85～92% 不透明度做主标题，副文字再降到 ~50～60%。dowse 之前 `row-name` 用了
`font-weight: 600`（Semibold），比 Raycast 实际用的 500 重一档，视觉上更"墩"更黑。
本次：标题 600→500，placeholder 不透明度从纯 `--fg-tertiary` 降到前景色的 40～50%。

## 五、列表行解剖

```
┌──────────────────────────────────────────────┐
│  [22px 图标]   标题文字（Medium 500）           │
│               路径/副文字（Regular，淡）        │  ← 选中/hover 时纯色块背景，无边框
└──────────────────────────────────────────────┘
                                        类型提示 →  （右侧对齐，独立一列，不挤标题）
```

- 左图标：20～24px，垂直居中对齐标题那一行（不是居中整个行块）。
- 标题 + 副文字：左对齐纵向堆叠，紧凑（2px 间距）。
- 右侧类型/分类提示：与图标同一行水平居中，右对齐，颜色最淡，字号最小。
- 命中态：整行一个纯色圆角块，块内无二次边框。Raycast **不给选中行加描边**，
  纯靠背景色反差。dowse 之前选中态是 `background + border` 双重强调，本次去掉描边，
  只留背景色块（更贴近 Raycast，也更轻）。

## 六、底部操作条解剖

```
┌──────────────────────────────────────────────────────────┐
│  ● 品牌小图标  名字        │        [↵ 主操作]  │  [⌘K 更多操作]  │
└──────────────────────────────────────────────────────────┘
```

- 左侧：一个小品牌标（Raycast 用自己的雷标；dowse 换成错位圆小图标 + "dowse"）。
- 右侧：kbd 键帽成组出现，组间用一条极细的竖分隔线或纯间距分开，主操作
  （回车）总是最靠右/最醒目，次要操作在左侧或收进"更多操作"里。
- 键帽本身：深色圆角小方块，`rgba(255,255,255,0.1)` 量级背景，文字用等宽感字体
  或至少数字对齐的字体特性。

dowse 现状：ShortcutBar 已经是"左品牌区（缺失）+ 右 kbd 提示"的骨架，但没有左侧品牌区。
本次补上左侧 dowse 错位圆图标 + "dowse" 字样，高度 32px→40px，
kbd 键帽从纯背景块继续沿用（已经对）。

## 七、选中态样式

浅色圆角块、无边框，见第五节。颜色上 Raycast 用的是极淡的白色叠加
（约 6～8% 不透明度），不用品牌色。**dowse 反过来：选中态用水蓝 accent 而不是中性色**，
这是刻意保留的身份差异点（任务书明确要求 accent 用水蓝，不用 Raycast 的红/橙）。
但要把水蓝的强度往 Raycast 的"轻"靠：`--accent-soft` 不透明度降一档、去掉描边。

## 八、动效节奏目测值

| 动作 | 时长 | 曲线 |
|------|------|------|
| 面板呼出 | ~120ms | 轻弹簧，几乎不觉察 |
| 列表行错峰入场 | 每行 ~10～15ms 错峰，单行 ~120ms | 轻弹簧 |
| 选中态切换 | ~90～100ms | ease-out，跟手不拖沓 |
| hover 背景淡入 | ~80～100ms | ease-out |

dowse 现状（`playShowAnimation` 120ms spring bounce 0.2、`ResultList` stagger 12ms、
`.row` transition 90ms）已经对齐这个节奏，本次不动画效部分，只动视觉参数。

## 九、本次三项硬约束改动汇总

| 项 | 改前 | 改后 |
|------|------|------|
| 暗色玻璃 alpha | `rgba(20,20,24,0.55)` | `rgba(18,18,22,0.32)` |
| 亮色玻璃 alpha | `rgba(255,255,255,0.65)` | `rgba(255,255,255,0.42)` |
| 窗口尺寸 | 720×480 | 750×500 |
| 行标题字重 | 600 | 500 |
| placeholder 不透明度 | 100%（`--fg-tertiary` 直接用） | 前景色 45% |
| 正文等宽字重 | 系统默认（Cascadia Mono 常规） | 不变，但预览正文改用更轻的字重档 |
| 底部操作条高度 | 32px | 40px |
| 选中行描边 | 有（`--accent-border` 1px） | 无（纯色块） |
