# 里程碑 2 设计说明：浮窗

这份文档定的是浮窗的交互、视觉和技术架构。实现照这份来，有冲突以这份为准；
要偏离先回来改文档。

## 一、这个窗口是什么

一个按快捷键随时呼出的搜索面板，长在系统正中偏上的位置。它不是一个"应用窗口"，
更像系统能力的延伸，参照 macOS Spotlight 和 Raycast 的定位。

```
                        ┌────────────────────────────────────────────┐
                        │  🔍  限流                                   │   ← 大输入框
                        ├───────────────────────────┬────────────────┤
                        │ ▸ 限流方案对比.md          │                │
                        │   D:\notes\arch\          │   预览区        │
                        │   …分布式[限流]器的令牌桶…  │                │
                        │ ─────────────────────────│  命中段落上下文  │
                        │   rate-limiter.rs         │  或图片缩略图    │
                        │   D:\code\dowse\          │                │
                        │   …[限流]中间件的实现…      │                │
                        ├───────────────────────────┴────────────────┤
                        │ ↵ 打开   ^↵ 打开所在文件夹   ^C 复制路径      │   ← 快捷键提示条
                        └────────────────────────────────────────────┘
```

三块结构：输入框、结果列表（左）、预览区（右）。底部一条快捷键提示，永远可见，
新用户看一眼就会用，老用户当它不存在。

## 二、交互规则

### 呼出与隐藏
- 全局快捷键 `Alt+\`` 呼出；`Esc`、点击窗口外、再按一次快捷键都是隐藏。
- **进程常驻，窗口只是 show/hide**。呼出必须感觉是"瞬间"，预算 50ms 以内。
  每次呼出时输入框自动聚焦、上次的查询词全选（直接打字即覆盖，想续用按方向键）。
- 开机自启（可在托盘菜单关掉），托盘图标常驻，右键菜单：呼出 / 重建索引 / 退出。

### 输入即搜
- 键入后 30ms 防抖发起搜索，键入到结果上屏预算 80ms。
- 查询词之间默认 AND 语义（tantivy 默认是 OR，要显式改掉，多词收窄结果
  才符合直觉）。带引号的短语查询保持原样支持。
- 空输入时显示引导文案；索引还没建过时，显示"选个目录开始建索引"的引导按钮。

### 键盘走天下
| 按键 | 行为 |
|------|------|
| ↑ / ↓ | 移动选中项，预览区跟随刷新 |
| Enter | 用系统默认程序打开选中文件 |
| Ctrl+Enter | 打开所在文件夹并选中该文件（explorer /select） |
| Ctrl+C | 复制完整路径 |
| Esc | 隐藏浮窗 |

鼠标全程可用但永远不是必需的。

### 结果行的信息构成
每行三层：文件类型图标 + 文件名（命中词高亮）、完整路径（灰色小字）、
命中摘要一行（命中词高亮）。预览区显示更长的命中上下文；图片文件显示缩略图。

## 三、视觉基调

一句话：**Win11 原生玻璃 + 苹果式克制**。

- **材质**：窗口本体用系统 Acrylic（亚克力磨砂），由 DWM 合成。透明必须是
  系统给的，前端画假磨砂一眼廉价。降级链：Acrylic → Mica → 纯色深灰，
  按系统版本和用户设置自动落，托盘菜单里给个"关闭透明效果"的开关。
- **圆角与阴影**：用系统原生的，不自己画。无标题栏、无边框按钮，这不是应用窗口。
- **字体**：正文内嵌 Inter（拉丁）+ MiSans（中文）；等宽场景（路径、代码正文）用系统自带的 Cascadia Mono，不内嵌。
- **主题**：明暗跟随系统，暗色是主打脸面（玻璃在暗色下最出效果），两套都要精修。
- **动画**：弹簧物理，不是缓动曲线。呼出＝轻微放大（0.98→1）+淡入，全程 120ms
  以内；结果列表变化时行元素做极短的错峰滑入；选中态切换要"跟手"。
  原则：动画是手感不是表演，任何一处超过 150ms 就是做错了。
- **信息密度**：宁可少显示，不可挤。行高、留白参照 Raycast 的呼吸感，
  反面教材是塞满屏幕的传统 Windows 工具风。

## 四、技术架构

```
crates/
├── dowse-core      已有：索引 + 搜索（本里程碑加：默认 AND、按路径取预览上下文）
├── dowse-cli       已有：命令行入口，保持可用
└── dowse-app       新增：Tauri 2 应用
    ├── src-tauri   Rust 侧：窗口管理、全局快捷键、托盘、调 dowse-core
    └── ui          前端：Svelte 5 + TypeScript + motion（动画库）
```

- **前后端在同一进程**：前端通过 Tauri command 调 Rust 侧，Rust 侧直接以库依赖
  的方式调 dowse-core 查索引。没有网络、没有进程间通信，这是 80ms 预算的底气。
- 窗口属性：透明、无装饰、不进任务栏、显示时置顶居中。
- 全局快捷键、托盘、打开文件/文件夹用 Tauri 官方插件，不自己造。
- 玻璃材质用 Tauri 的 window effects 能力（背后是 DWM 的 backdrop API）。
- 前端不引 UI 组件库。这个界面总共没几个组件，手写才能控制质感；
  只引 motion 做弹簧动画。

## 五、性能预算（超了就是 bug）

| 指标 | 预算 |
|------|------|
| 快捷键按下到窗口可见 | < 50ms |
| 键入到结果上屏 | < 80ms |
| 常驻内存（含索引 reader） | < 150MB |
| 安装包 | < 15MB（2026-07 修订为 < 50MB：本指标定于 GUI 单体时期，0.8.3 起安装包捆绑 CLI sidecar，旧预算已不合时宜） |

## 六、验收清单

1. Alt+` 呼出，感觉不到等待；Esc 隐藏；重按呼出键切换显隐。
2. 输入"限流"，结果实时出现，文件名和摘要里的命中词高亮。
3. 全键盘完成一次：搜索 → 选中 → Ctrl+Enter 跳到文件夹看到文件被选中。
4. 暗色/亮色主题下玻璃效果都成立；关闭透明后依然好看。
5. 断网、索引为空、目录被删三种情况都有得体的界面反馈，不白屏不报错弹窗。
6. 性能预算四项全部达标（附实测数字）。

## 七、明确不做（本里程碑）

- 文件监听增量索引（里程碑 3）、OCR（里程碑 4）、MCP（里程碑 5）。
- 设置界面：所有配置暂时走托盘菜单和配置文件。
- 多显示器记忆位置、搜索历史、插件系统：都先不做。
