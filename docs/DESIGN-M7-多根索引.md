# 多根索引设计说明

目标一句话：同时索引多个文件夹（文档盘 + 截图目录 + 工作区），可单独增删，
不再是"换一个就丢一个"的单根世界。

## 一、现状与差距

基础设施其实早已多根就绪：meta.json 的 roots 本来就是列表、监听（watch_roots_auto）
和启动对账天生吃多根、搜索层根本不关心根。真正缺的只有两段：

1. **增量地"加一个根"**——现在唯一的建索引入口是全量重建（删库重来），
   加第二个根会把第一个根抹掉；
2. **移除一个根**——从索引里删掉该根全部文档并停掉它的监听。

## 二、核心操作语义

| 操作 | 实现 | 复用 |
|------|------|------|
| 添加根 | 不动现有索引，对新根做一次"目录树 upsert"（遍历 + 先删后加，幂等），完成后 roots 追加、监听热挂载 | 增量更新器的 UpsertTree 路径、OCR 队列照常吃新图片 |
| 移除根 | path 前缀圈选删除该根全部文档 + OCR 队列 compact 掉该根条目 + roots 移除 + 监听卸载 | RemoveTree 的前缀删除、既有 compact |
| 重建单根 | 移除根 + 添加根 的组合 | 全部复用 |
| 全量重建 | 保留现语义（删库、按 roots 逐根重建）——schema 升级时的通道 | 现有 rebuild |

约束：**根之间不允许嵌套**（A 包含 B 时拒绝添加并提示，含双向检查）——嵌套会让
前缀删除和对账语义变成泥潭，产品上也没有正当需求。路径比较用规范化后的形态。

## 三、界面（延续"几乎不存在"原则）

- 托盘"索引文件夹"子菜单：每根一项（显示路径 + 文档数），子项：重建 / 移除；
  末尾"添加文件夹…"。加/删期间相关项置灰防重入。
- 浮窗空态：列出全部根（每行一条灰字小字），下方保留"添加文件夹"的小字入口。
- 进度直播复用现有整套（v0.6.1 的随时可见机制），加根时直播的是该根的增量进度。

## 四、边界与失败

- 添加中途退出：目录树 upsert 幂等,下次启动对账自动补齐——meta 的 roots
  在 upsert **完成后**才写入,半程崩溃不留幽灵根。
- 移除中途退出：roots 先移除再删文档——残留文档由"对账发现不在任何根内→删除"
  兜底（对账需加这条规则：索引中文档的 path 不属于任何注册根时删除）。
- 根所在盘符拔掉/网络盘失联：对账跳过不可达根并日志说明,不误删（现有行为核验）。

## 五、验收清单

1. 加根 B 后：A、B 内容都可搜；B 的图片进 OCR 队列；改 B 下文件秒级可搜。
2. 移除 B：B 内容全部消失，A 不受影响，OCR 队列无 B 残留。
3. 尝试添加 A 的子目录 / 父目录：拒绝且提示清晰。
4. 加根中途杀进程重启：对账后 B 完整可搜，无重复文档。
5. 托盘与空态的根列表实时反映增删。

## 六、明确不做

- 每根独立的包含/排除规则（glob 过滤）——等真实需求
- 根级别的启停开关——移除已经够用
- 跨根去重（同一文件被两个根经硬链接覆盖）——罕见,不值当
